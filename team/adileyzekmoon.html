Enhancement Added: Filter/FilterAll

External behavior

Start of Extract [from: User Guide]

Locating persons by any parameter: filter (alias fi)

Finds persons who has parameters that matches the given keyword.
Format: filter KEYWORD [MORE_KEYWORDS] or fi KEYWORD [MORE_KEYWORDS]

  • The search is case insensitive. e.g friends will match Friends

  • The order of the keywords does not matter. e.g. Alex CS2103 Groupmate will match Groupmate CS2103 Alex

  • All parameters are searched.

Examples:

  • filter friends
    Returns Alex Yeoh and Bernice Yu

  • filter colleagues
    Returns Bernice Yu and Roy Balakrishnan

  • fi alex Colleagues
    Returns Alex Yeoh, Bernice Yu and Roy Balakrishnan

Locating persons by all parameter: filterall (alias fa)

Finds persons who has parameters that matches the given keyword.
Format: filterall KEYWORD [MORE_KEYWORDS] or fa KEYWORD [MORE_KEYWORDS]

  • The search is case insensitive. e.g friends will match Friends

  • The order of the keywords does not matter. e.g. Alex CS2103 Groupmate will match Groupmate CS2103 Alex

  • All parameters are searched.

Examples:

  • filterall friends
    Returns Alex Yeoh and Bernice Yu

  • filterall colleagues
    Returns Bernice Yu and Roy Balakrishnan

  • fa Bernice Colleagues
    Returns Bernice Yu

  • fa colleagues friends
    Returns Bernice Yu

End of Extract

Justification

The user will be able to search for specific contacts or a range of contacts easily.

Implementation

Start of Extract [from: Developer Guide]

Filter feature implementation

The FilterCommand and FilterAllCommand serve very similar functions to the FindCommand and FindTagCommand. However, they differ in the way that they allow more flexcibility to the user in terms of filtering contacts because:

  • All parameters are included in the seraching implementation

  • Users are able to utilise OR or AND logic to make the searcha s narrow or wide as desired.

    • This eliminatest the drawbacks of previous FindCommand and FindTagCommand implementations.

Sequence diagram can be inferred from FindTagCommand sequence diagram above.

AS shown above, FilterCommand will output a list of contacts that match any of the given keywords, while FilterAllCommand will output only contacts that match all keywords.

Design Considerations

Aspect: how filter implementation is executed (AND logic vs OR logic)
Alternative 1 (FilterCommand): Utilising OR logic
Pros: More flexible usage allows user to view contacts across various parameters ,
user does not have to utilise more than 1 command to view various contacts.
user is able to searcha wide array of contacts with one command Cons: Looking for specific contacts by using more than one keyword is not possible.
Alternative 2 (FilterAllCommand): Utilising AND logic
Pros: Allows finding of specific contacts with more than 1 tag.
Cons: Limits flexibility for power users that wish to view various contacts across more than 1 tag.

End of Extract

Enhancement Added: Social Media (Facebook)

External behavior

Start of Extract [from: User Guide]

Social Media link

Selecting a contact brings you directly to a list of Facebook profiles that match his/her name.

End of Extract

Justification

The user wants to visit his contacts' social media pages easily.

Implementation

Start of Extract [from: Developer Guide]

Social Media link (Facebook)

The BrowserPanel java file has been enhanced to display a Facebook user search of the selected contact’s name rather
than a Google search.

  • The search includes the full name of the contact.

Design Considerations

Aspect: How the Facebook webpage is loaded
Alternative 1 (current choice): Utilising a Facebook user search
Pros: Little chance of loading a wrong page, as the user is able to select the proper profile
page from a list of closest matches. Cons: Will have to go throught the extra steps of locating and selecting a profile page
Alternative 2 : Loading a Facebook profile page that matches the selected contact’s name
Pros: Direct access to the selected contact’s profile page for easy use.
Cons: Extremely likely to load the wrong profile page, due to overlapping names of millions of Facebook accounts.

End of Extract
